# Opus: REST API Server (FastAPI)

## 📌 Назначение
Этот микросервис выступает слоем бизнес-логики между Фронтендом (`/web`) и Базой Данных Supabase. Он обрабатывает регистрацию, логин работодателей, обеспечивает доступ к списку сотрудников (паблик/приват часть), списание "карточек" за просмотры и получение тарифов.

## 🛠 Используемые технологии
- **Фреймворк:** FastAPI + Uvicorn
- **Валидация данных:** Pydantic
- **Интеграция с БД:** Использование `supabase` Python клиента:
  - Существуют два инстанса Supabase клиента (см. `database.py`):
    - `get_service_client()`: Для админских задач (например, обновление чужих записей подписок или изменение лимитов).
    - `get_anon_client()`: Для пользовательских запросов (работает поверх RLS, если передать JWT токен пользователя).
- **Аутентификация:** Middleware `get_current_user` (`middleware/auth.py`), который проверяет Bearer токен в Supabase Auth.

## 📂 Архитектура (папка `/routers`)
- `auth.py`: Роуты `/api/auth/register`, `/login`, `/reset-password`. Обращаются напрямую к встроенному в Supabase Auth сервису (GoTrue).
- `employees.py`: Роуты:
  - `GET /` - Получение списка кандидатов с пагинацией (Доступно всем публично).
  - `GET /count` - Подсчет общего количества карточек (для списка).
  - `POST /{employee_id}/view` - Запрос на "открытие" контактов. Внутри происходит проверка наличия подписки и уменьшение `cards_remaining` на 1 (логика списания).
- `payments.py`: Заглушка (в будущем) для Fenik Pay. Создает Payment Record в базе и активирует `subscription`.
- `tariffs.py`: Вывод списка тарифов и текущей подписки работодателя.

## 🚀 Запуск для ИИ Агентов
1. `pip install -r requirements.txt`
2. Настройте файл `.env`.
3. Копировать `.env.example` -> `.env`.
4. Запустите: `uvicorn main:app --reload`. API документация Swagger автоматическая (доступна на `http://localhost:8000/docs`).
