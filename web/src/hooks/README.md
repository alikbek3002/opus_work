# React React-Query Хуки (Hooks)

Эта папка содержит кастомные React-хуки, в которых сосредоточена вся бизнес-логика (работа с сервером) фронтенда.

## 🛠 Технологии
- `@tanstack/react-query` (v5+) — стейт-менеджер серверных данных (caching, deduplication, invalidation).
- Наш `lib/api.ts` выступает как HTTP-провайдер.

## 📂 Файлы и их назначение
1. **`useAuth.ts`**: Обработка авторизации (`login`, `register`, `logout`). 
   - Вызывает методы `api.ts`, при успехе сохраняет `access_token` в `localStorage`.
   - ВАЖНО: При `logout` он делает `queryClient.clear()`, чтобы чужие данные стерлись!
2. **`useEmployees.ts`**: Работа с карточками сотрудников.
   - `useEmployees()`: скачивает базовый публичный список с бэкенда.
   - `useViewEmployee()`: это **`useMutation`**. Инициирует списывание "лимита карточек" и инвалидирует кэш профиля.
3. **`useTariffs.ts`**: 
   - `useTariffs()`: качает инфу по прайсам.
   - `useSubscription()`: отслеживает, сколько "Просмотров" (карточек) осталось у текущего работодателя (`cards_remaining`). Когда происходит `createPayment` — он перезапрашивается (инвалидация).

## 💡 Для ИИ Агентов
React Query работает так, что он делает умные ретраи и держит в кэше `staleTime`.
- Не делайте запросы в `useEffect`. Вместо этого: пишите новый хук `useQuery` в этой папке, если нужны новые данные с бэкенда.
- В `lib/query.ts` лежат ключи запросов (`queryKeys`), используйте их, чтобы по ключу инвалидировать (обновлять) данные.
