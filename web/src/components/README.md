# React Components (UI & Smart)

Эта папка содержит переиспользуемые строительные блоки пользовательского интерфейса.

## 🛠 Технологии
- `React 19` (Functional Components)
- `Tailwind CSS` & `clsx`, `tailwind-merge` для работы с классами.
- `Radix UI` — примитивы для доступности (Accessibility) базовых элементов (Dialog, Tabs, Checkbox и т.д.).
- `Framer Motion` — анимации, эффекты ховера, появление elementów (используется в `EmployeeCard.tsx`).

## 📂 Структура и назначение
1. **`/ui`** — Папка с "глупыми" (базовыми) компонентами, сгенерированными через shadcn/ui.
   - `button.tsx`, `input.tsx`, `dialog.tsx`, `card.tsx` и др.
   - *Для ИИ Агентов:* Если нужна новая кнопка или инпут, используйте эти готовые компоненты, а не создавайте с нуля через `div` и `button`.

2. **Смарт-компоненты (Корень `/components/`)**:
   - `EmployeeCard.tsx`: Карточка сотрудника для `Dashboard`. Обернута в `framer-motion` (анимация появления и hover). Кнопка "Просмотреть" запускает `onView`, проверяет токен и списывает баланс.
   - `LoginModal.tsx`: Отвечает за логин и регистрацию "на лету" (без ухода со страницы). Слушает React стейт и управляет автозаполнением пароля браузера.
   - `SettingsModal.tsx`: Отвечает за отображение email и кнопку сброса пароля.
   - `TariffCard.tsx`: Визуальная карточка для страницы `Tariffs` (с кнопкой "Купить").

## 💡 Полезно знать
Интерактивные элементы (Модалки) часто управляются через `useState` в родительских компонентах (в `Layout.tsx`), а также слушают глобальные события `window.dispatchEvent(new Event('openLoginModal'))`, если `api.ts` поймал `401 Unauthorized`.
