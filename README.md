# Opus Platform (Telegram Bot + FastAPI + React)

## 📌 Описание проекта
Opus — это комплексная платформа для поиска сотрудников. Проект состоит из трех основных частей:
1. **Telegram-бот** — для регистрации резюме сотрудников-кандидатов (сбор данных: район, специализация, возраст, опыт работы).
2. **FastAPI Бэкенд** — бизнес-логика, управление тарифами (работодатели покупают возможность открывать контакты кандидатов), интеграция с Finik Acquiring и авторизация.
3. **React-фронтенд (Web)** — дашборд для работодателей, где можно фильтровать базу кандидатов, покупать подписку/тарифы и просматривать полные профили сотрудников при наличии активного тарифа.
4. **Supabase** — централизованная база данных (PostgreSQL) + система авторизации (Supabase Auth).

## 🛠 Общий технологический стек
- **Database / Auth:** Supabase (PostgreSQL, Row Level Security, JWT Auth)
- **Схема БД скрипты:** `supabase/migrations/001_init.sql` (Все таблицы и RLS политики)
- **Бот:** Python, `python-telegram-bot`, `supabase-py`
- **Сервер (API):** Python, `FastAPI`, `Pydantic`, `Uvicorn`
- **Фронтенд:** TypeScript, `React`, `Vite`, `React Query (TanStack)`, `Tailwind CSS`, `Framer Motion`, `Radix UI (shadcn)`

## 📂 Структура проекта
- `/bot` — Исходный код Telegram-бота (Python)
- `/server` — Исходный код REST API приложения (FastAPI)
- `/web` — Исходный код веб-платформы для работодателей (React)
- `/supabase` — Миграции базы данных и конфигурации

## 🚀 Детали запуска
Каждый микросервис содержит свой собственный `README.md` файл с инструкциями по установке и настройке переменных окружения (`.env`).
Все они должны быть запущены одновременно для полноценной работы экосистемы.
